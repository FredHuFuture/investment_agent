<design_decisions>

**1. Single page (not two):** Combine LIVE-02 (calibration table) + LIVE-03 (weights editor) into ONE `CalibrationPage.tsx` with two sections stacked vertically. Per planning context hint: "recommend embedded section for simpler nav." Rationale: both surfaces are consulted together in the weekly review workflow, and it halves the number of routes to maintain. The legacy `/weights` route redirects to `/calibration` with an anchor (or is rewritten to host the new content too).

**2. Sparkline strategy:** `/analytics/calibration` does NOT currently return a rolling IC time series — it returns a single `ic_5d` number per agent. The sparkline needs a time series. **Decision:** the sparkline renders the `rolling` list IF present on the response (extend `api/routes/calibration.py` in a minor extension — see Task 2); otherwise renders a single-point fallback ("historical IC data accumulating…"). The extension: add `rolling_ic: list[number | null]` to each agent in the response. This is a small backend extension piggybacking on the existing endpoint — ~15 lines in `api/routes/calibration.py` — scoped to this plan because it's frontend-driven and trivial.

**Color coding for sparkline (from context):**
- IC-IR > 1.0 → green (`stroke="#10B981"`)
- 0.5 ≤ IC-IR ≤ 1.0 → amber (`stroke="#F59E0B"`)
- IC-IR < 0.5 (including negative) → red (`stroke="#EF4444"`)
- IC-IR null → gray (`stroke="#6B7280"`)

**3. Legacy `WeightsPage.tsx` handling:** The v1.0 donut WeightsPage.tsx consumes `WeightsData` with `buy_threshold/sell_threshold/crypto_factor_weights` which 06-01 dropped. Rewrite WeightsPage.tsx as a redirect: `<Navigate to="/calibration#weights" replace />`. No more donut. This is acceptable scope-wise because the NEW weights UI is richer. Add a brief changelog note in SUMMARY.md so the user knows the old donut is gone.

**4. `AssetTypeTabs` state:** Local React state (`useState<"stock"|"btc"|"eth">("stock")`). Not routed via URL because the asset-type selection only matters for the weights section; the calibration table rows are asset-agnostic (agent IC/Brier are global across asset types per the current backend implementation).

**5. Empty-corpus CTA:** When `corpus_metadata.total_observations === 0` OR all agents have `sample_size === 0`:
- Hide the table
- Show: "Corpus is empty. To see calibration metrics, populate it by clicking below."
- Button: "Rebuild corpus" → POST `/api/v1/analytics/calibration/rebuild-corpus` with empty body → toast + poll `/{job_id}` until status != 'running'; on complete, refetch calibration
- This closes the planner context requirement about "empty-corpus graceful degradation"

**6. data-testid namespace:** All new components use `data-testid="cal-..."` prefix (e.g., `cal-agent-row-TechnicalAgent`, `cal-ic-sparkline-TechnicalAgent`, `cal-apply-ic-ir`, `cal-exclude-toggle-stock-SentimentAgent`, `cal-asset-type-tab-stock`).

**7. No new npm packages:** Recharts 2.15.4 already installed. Custom SVG for the tab underline (like DailyPnlHeatmap). `@testing-library/user-event` already available for click tests.

**8. Sidebar navigation entry:** Add `"Calibration"` menu item to `frontend/src/components/layout/Sidebar.tsx` — verify existing convention first, add the same way.
</design_decisions>

    function fmtPct(v: number | null | undefined): string {
      if (v === null || v === undefined) return "--";

    }

    function fmtDelta(current: number | undefined, suggested: number | null | undefined): string {
      if (current === undefined || suggested === null || suggested === undefined) return "--";
      const delta = (suggested - current) * 100;
      const sign = delta >= 0 ? "+" : "";
      return `${sign}${delta.toFixed(1)}pp`;
    }

      onOverride,

    import { useToast } from "../contexts/ToastContext";

    export default function CalibrationPage() {
      usePageTitle("Calibration");
      const { toast } = useToast();
      const [assetType, setAssetType] = useState<AssetType>("stock");
      const [applying, setApplying] = useState(false);
      const [rebuilding, setRebuilding] = useState(false);

      const calApi = useApi<CalibrationResponse>(
        () => getCalibrationAnalytics(),
        { cacheKey: "cal:calibration", ttlMs: 60_000 },
      );
      const weightsApi = useApi<WeightsOverviewResponse>(
        () => getWeightsV2(),
        { cacheKey: "cal:weights", ttlMs: 60_000 },
      );

      const refetchAll = useCallback(() => {
        calApi.refetch();
        weightsApi.refetch();
      }, [calApi, weightsApi]);

      async function handleApplyIcIr() {
        setApplying(true);
        try {
          await applyIcIrWeights();
          toast({ type: "success", message: "IC-IR weights applied" });
          weightsApi.refetch();
        } catch (err) {
          const msg = err instanceof Error ? err.message : "Failed to apply IC-IR weights";
          toast({ type: "error", message: msg });
        } finally {
          setApplying(false);
        }
      }

      async function handleOverride(agent: string, excluded: boolean) {
        try {
          await overrideAgentWeight({ agent, asset_type: assetType, excluded });
          toast({
            type: "success",
            message: `${agent} ${excluded ? "excluded" : "enabled"} for ${assetType}`,
          });
          weightsApi.refetch();
        } catch (err) {
          const msg = err instanceof Error ? err.message : "Override failed";
          toast({ type: "error", message: msg });
        }
      }

        setRebuilding(true);

          toast({ type: "error", message: msg });
        }

      const error = calApi.error || weightsApi.error;

              Refresh
            </Button>
          </div>

          {warnings.length > 0 && <WarningsBanner warnings={warnings} />}
          {error && <ErrorAlert message={error} onRetry={refetchAll} />}

          {loading && (
            <div className="space-y-6">

              <SkeletonCard className="h-[280px]" />
            </div>
          )}

          {!loading && calApi.data && (
