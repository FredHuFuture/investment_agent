<objective>
LIVE-03 backend: add a persisted `agent_weights` table (source of truth for production signal aggregation), expose three HTTP endpoints (`GET /weights` overview, `POST /weights/apply-ic-ir` to persist IC-IR-suggested weights, `PATCH /weights/override` to exclude/renormalize an agent), and wire `SignalAggregator` to load from this table — so Phase 6 frontend can display, apply, and override weights, and the next daemon `signal_aggregator` run uses the persisted values.

Purpose: Makes the signal-aggregator weights user-actionable and persistent. Today weights live only in `portfolio_meta.adaptive_weights` JSON blob (via WeightAdapter.save_weights) which is per-asset-type dict and cannot express per-agent manual overrides. The new table splits rows per-(agent, asset_type) so the UI can toggle a single agent off without touching the rest.

Output: 1 new DB table, 3 new endpoints, 2 Pydantic models, 1 aggregator helper, 1 test file. Backend-only. Phase 6 frontend (06-02) consumes this surface.
</objective>

<design_decisions>
**1. Schema for `agent_weights` (NEW table):**

```sql
CREATE TABLE IF NOT EXISTS agent_weights (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    agent_name TEXT NOT NULL,
    asset_type TEXT NOT NULL CHECK (asset_type IN ('stock', 'btc', 'eth')),
    weight REAL NOT NULL DEFAULT 0.0,
    manual_override INTEGER NOT NULL DEFAULT 0,   -- 0/1 boolean
    excluded INTEGER NOT NULL DEFAULT 0,           -- 0/1 boolean; takes precedence over weight
    source TEXT NOT NULL DEFAULT 'default'
        CHECK (source IN ('default', 'ic_ir', 'manual')),
    updated_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE (agent_name, asset_type)
);

CREATE INDEX IF NOT EXISTS idx_agent_weights_asset_type ON agent_weights (asset_type);
```

Seed behavior: when `init_db` sees the table EMPTY, seed rows from `SignalAggregator.DEFAULT_WEIGHTS` with `source='default'`. Never overwrite on subsequent startups.

**2. Endpoint response shapes (exact JSON):**

`GET /weights` returns:
```json
{
  "data": {
    "current": {
      "stock": {"TechnicalAgent": 0.25, "FundamentalAgent": 0.40, ...},
      "btc":   {"CryptoAgent": 0.80, "TechnicalAgent": 0.20},
      "eth":   {"CryptoAgent": 0.80, "TechnicalAgent": 0.20}
    },
    "suggested_ic_ir": {
      "stock": {"TechnicalAgent": 0.32, ...} | null,
      "btc":   {...} | null,
      "eth":   {...} | null
    },
    "overrides": {
      "stock": {"SentimentAgent": {"excluded": true, "manual_override": true}},
      "btc":   {},
      "eth":   {}
    },
    "source": "default" | "ic_ir" | "manual",
    "computed_at": "2026-04-23T…Z",
    "sample_size": 0
  },
  "warnings": []
}
```

`suggested_ic_ir` is `null` (not empty dict) when `compute_ic_weights` returns None (corpus empty). The frontend uses null to gate the "Apply IC-IR weights" button.

`POST /weights/apply-ic-ir` (no body) returns:
```json
{
  "data": {
    "applied": true,
    "weights": { /* same shape as current above */ },
    "source": "ic_ir",
    "sample_size": 240
  },
  "warnings": []
}
```
Returns 409 `{code: "NO_IC_IR_DATA", message: "..."}` when `compute_ic_weights` returns None.

`PATCH /weights/override` body `{agent: str, asset_type: str, excluded: bool}` returns:
```json
{
  "data": {
    "agent": "SentimentAgent",
    "asset_type": "stock",
    "excluded": true,
    "manual_override": true,
    "renormalized_weights": {"TechnicalAgent": 0.294, "FundamentalAgent": 0.471, ...},
    "source": "manual"
  },
  "warnings": []
}
```

**3. Backward compatibility of existing GET /weights:**
Existing consumers (frontend `WeightsPage.tsx` v1.0 donut) expect the OLD shape `{weights, crypto_factor_weights, buy_threshold, sell_threshold, source, sample_size}`. We keep the old behavior alive but change the contract: the new GET /weights returns the new shape. The old donut WeightsPage will break visually; that is acceptable because plan 06-02 replaces it. Note in SUMMARY.md.

**4. Aggregator wiring (`engine/aggregator.py`):**
Add module-level async helper `load_weights_from_db(db_path)` returning `dict[str, dict[str, float]] | None`. It reads `agent_weights` rows, skips `excluded=1`, renormalizes each asset_type to sum 1.0 per FOUND-05, returns None if the table is empty. NOT called from `__init__` (constructor stays sync); call sites that want DB weights invoke the helper and pass the result as the `weights=` kwarg. The `pipeline.py` and daemon call sites are out of scope here — this plan only lands the helper + tests; wiring pipeline comes in a later plan if needed.

**5. Deferred import contract:**
Import `compute_ic_weights` inside the endpoint body, not at module top. Prevents circular imports between `api.routes.weights` ↔ `engine.weight_adapter` ↔ `engine.aggregator`.

    ```python
    async def load_weights_from_db(db_path: str) -> dict[str, dict[str, float]] | None:
        """Load agent weights from the `agent_weights` table (LIVE-03).

        - Skips rows where excluded=1.
        - Renormalizes each asset_type so remaining weights sum to 1.0 (FOUND-05 contract).
        - Returns None when the table is empty (caller falls back to DEFAULT_WEIGHTS).

        Deferred import of aiosqlite avoids making aggregator a DB-dependent module at
        import time — tests and offline tooling that construct SignalAggregator directly
        should not need aiosqlite on the path.
        """
        import aiosqlite  # deferred
        async with aiosqlite.connect(db_path) as conn:
            conn.row_factory = aiosqlite.Row
            rows = await (
                await conn.execute(
                    """
                    SELECT agent_name, asset_type, weight, excluded
                    FROM agent_weights
                    """
                )
            ).fetchall()
        if not rows:
            return None
        raw: dict[str, dict[str, float]] = {}
        for row in rows:
            if int(row["excluded"]) == 1:
                continue
            raw.setdefault(row["asset_type"], {})[row["agent_name"]] = float(row["weight"])
        # Renormalize each asset_type to sum 1.0 (FOUND-05 contract).
        normalized: dict[str, dict[str, float]] = {}
        for asset_type, agents in raw.items():
            total = sum(agents.values())
            if total > 0:
                normalized[asset_type] = {k: round(v / total, 4) for k, v in agents.items()}
            else:
                normalized[asset_type] = agents
        return normalized if normalized else None
    ```

    async def _read_current_weights(conn) -> tuple[
        dict[str, dict[str, float]],
        dict[str, dict[str, dict[str, bool]]],
        str,

    ]:

        current_weights: asset_type -> {agent: weight}, EXCLUDES excluded=1 rows,

        ).fetchall()

        raw: dict[str, dict[str, float]] = {}
        overrides: dict[str, dict[str, dict[str, bool]]] = {
            at: {} for at in VALID_ASSET_TYPES
        }
        sources: set[str] = set()
        latest_updated_at = ""

        for r in rows:
            at = r["asset_type"]
            sources.add(r["source"])
            if r["updated_at"] > latest_updated_at:
                latest_updated_at = r["updated_at"]
            if int(r["manual_override"]) == 1 or int(r["excluded"]) == 1:
                overrides.setdefault(at, {})[r["agent_name"]] = {
                    "excluded": bool(r["excluded"]),
                    "manual_override": bool(r["manual_override"]),
                }
            if int(r["excluded"]) == 1:
                continue
            raw.setdefault(at, {})[r["agent_name"]] = float(r["weight"])

        # Renormalize each asset_type to sum 1.0 (FOUND-05 contract).
        current: dict[str, dict[str, float]] = {}
        for at, agents in raw.items():
            total = sum(agents.values())
            if total > 0:
                current[at] = {k: round(v / total, 4) for k, v in agents.items()}
            else:
                current[at] = agents

        # Dominant source: manual > ic_ir > default
        if "manual" in sources:
            source = "manual"
        elif "ic_ir" in sources:
            source = "ic_ir"
        else:
            source = "default"

        return current, overrides, source, latest_updated_at


    async def _compute_suggested_ic_ir(db_path: str) -> tuple[
        dict[str, dict[str, float] | None],
        int,
    ]:
        """Compute IC-IR-suggested weights via existing WeightAdapter.compute_ic_weights.

        Returns ({asset_type: weights_or_None}, total_sample_size). If compute_ic_weights
        returns None (corpus empty), all per-asset-type values are None.
        """
        from engine.weight_adapter import WeightAdapter  # deferred
        from tracking.store import SignalStore
        from tracking.tracker import SignalTracker

        store = SignalStore(db_path)
        tracker = SignalTracker(store)
        adapter = WeightAdapter(db_path=db_path)
        adaptive = await adapter.compute_ic_weights(
            tracker=tracker, asset_types=["stock", "btc", "eth"],
        )
        if adaptive is None:
            return ({at: None for at in VALID_ASSET_TYPES}, 0)
        suggested: dict[str, dict[str, float] | None] = {}
        for at in VALID_ASSET_TYPES:
            suggested[at] = adaptive.weights.get(at) or None
        return (suggested, adaptive.sample_size)

    async def get_weights(db_path: str = Depends(get_db_path)):
        """LIVE-03: Return current per-(agent,asset_type) weights, IC-IR suggestions, overrides."""
        async with aiosqlite.connect(db_path) as conn:

    @router.post("/weights/apply-ic-ir", response_model=ApplyIcIrResponse)
    async def apply_ic_ir_weights(db_path: str = Depends(get_db_path)):
        """LIVE-03: Persist IC-IR-suggested weights to agent_weights with source='ic_ir'.

        Returns 409 NO_IC_IR_DATA when corpus is empty.
        """
        suggested, sample_size = await _compute_suggested_ic_ir(db_path)
        if all(v is None for v in suggested.values()):
            raise HTTPException(
                status_code=409,
                detail={
                    "code": "NO_IC_IR_DATA",
                    "message": (
                        "IC-IR weights unavailable — backtest_signal_history is "
                        "empty or has insufficient rows. Populate the corpus first "
                        "via POST /analytics/calibration/rebuild-corpus."
                    ),
                },
            )

        now = datetime.now(timezone.utc).isoformat()
        async with aiosqlite.connect(db_path) as conn:
            # UPSERT each asset_type's agents with source='ic_ir'.
            # We ONLY overwrite rows where manual_override=0 — a user's explicit
            # override takes priority over machine-suggested values.
            for at, agent_map in suggested.items():
                if agent_map is None:
                    continue
                for agent_name, weight in agent_map.items():
                    await conn.execute(
                        """
                        INSERT INTO agent_weights
                            (agent_name, asset_type, weight, manual_override,
                             excluded, source, updated_at)
                        VALUES (?, ?, ?, 0, 0, 'ic_ir', ?)
                        ON CONFLICT (agent_name, asset_type) DO UPDATE SET
                            weight = excluded.weight,
                            source = excluded.source,
                            updated_at = excluded.updated_at
                        WHERE agent_weights.manual_override = 0
                        """,
                        (agent_name, at, float(weight), now),
                    )
            await conn.commit()
            current, _, source, _ = await _read_current_weights(conn)

        return ApplyIcIrResponse(
            applied=True,
            weights=current,
            source=source,
            sample_size=sample_size,
        )


    @router.patch("/weights/override", response_model=OverrideResponse)
    async def override_agent_weight(
        body: OverrideRequest,
        db_path: str = Depends(get_db_path),
    ):
        """LIVE-03: Toggle an agent's excluded flag (persists manual_override=1)."""
        if body.agent not in KNOWN_AGENTS:
            raise HTTPException(
                status_code=400,
                detail={
                    "code": "UNKNOWN_AGENT",
                    "message": f"Unknown agent: {body.agent!r}. Known: {sorted(KNOWN_AGENTS)}",
                },
            )
        # body.asset_type is already validated by Pydantic regex.

        now = datetime.now(timezone.utc).isoformat()
        excluded_int = 1 if body.excluded else 0

        async with aiosqlite.connect(db_path) as conn:
            # UPSERT: if row missing (new agent added post-seed), insert with default weight 0.0.
            await conn.execute(
                """
                INSERT INTO agent_weights
                    (agent_name, asset_type, weight, manual_override,
                     excluded, source, updated_at)
                VALUES (?, ?, 0.0, 1, ?, 'manual', ?)
                ON CONFLICT (agent_name, asset_type) DO UPDATE SET
                    manual_override = 1,
                    excluded = excluded.excluded,
                    source = 'manual',
                    updated_at = excluded.updated_at
                """,
                (body.agent, body.asset_type, excluded_int, now),
            )
            await conn.commit()
            current, _, _, _ = await _read_current_weights(conn)

        return OverrideResponse(
            agent=body.agent,
            asset_type=body.asset_type,
            excluded=body.excluded,
            manual_override=True,
            renormalized_weights=current.get(body.asset_type, {}),
            source="manual",
        )
